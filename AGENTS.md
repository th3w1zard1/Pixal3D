# Pixal3D Agent Notes

- Hosted Space recovery checklist and verification history: [docs/SPACE_RECOVERY.md](docs/SPACE_RECOVERY.md). When validating browser and CLI in one session, run the browser default-sample flow **before** `space_smoke.py --generate` so ZeroGPU quota is not consumed by the CLI run first.
- When a task touches Hugging Face Spaces, ZeroGPU, Gradio, `hf`, or browser-to-backend transport, look up current official docs with Context7 before changing code.
- For this repo's hosted ZeroGPU path, preserve browser calls through `@gradio/client` unless current docs explicitly say another browser transport is supported for `Server` apps on ZeroGPU.
- Do not add regression or unit tests for runtime or UI fixes in this repo unless the user explicitly asks for tests.
- Do not declare a hosted Space fix complete from source edits alone.
- If runtime-facing files changed and there is no conflicting unrelated local work in those same files, deploy the candidate change to the test Space with the `hf` CLI.
- After deploy, verify the live Space in a browser and run one end-to-end check with a default sample image before closing the task.
- **Pre-ship:** `./scripts/pre_ship.sh` runs `./scripts/workflow_hygiene.sh` then `./scripts/agent_gate.sh` (forwards `--parity`, `--url`, `--write-summary`). Index: `docs/workflow-hygiene.md`.
- **Workflow hygiene (static):** `./scripts/workflow_hygiene.sh` alone when you only need validators (no browser). Optional `--parity` adds `check_repo_parity.py`.
- **Post-recovery agent gate:** `./scripts/agent_gate.sh` (parity + health/HTML + browser E2E + JSON summary on stdout; progress on stderr). Parse with `./scripts/agent_gate.sh 2>/dev/null | jq -e .overall_ok`. Optional `--write-summary docs/gate-results/latest.json` to persist audit JSON (`schema_version: pixal3d-agent-gate/3`, includes `space_repo_git_head` and `adapter_policy_*` from live `/health`); validated with `scripts/validate_gate_json.py`. See `docs/gate-results/README.md`. Browser exit **1** with explicit ZeroGPU quota copy is a verified pass when `overall_ok` is true. Optional `--url URL`. For health/HTML only, use `./scripts/verify_hosted_space.sh`. Do not combine browser gate with `--generate` in the same session. For optional full generate smoke, create a venv and install `scripts/smoke-requirements.txt` first (`gradio_client` is not in the Space runtime requirements). Use `--generate` to call `/generate_3d` with ZeroGPU-safe defaults (512 / 5 steps); on hosted ZeroGPU the smoke skips `/warmup_runtime` (same as the browser UI). Anonymous cold runs usually take ~2–3 minutes and return a geometry-only `glb_path`; sign in on the Space for preview frames and textured extract. Optional `--write-manifest docs/generation-manifests/latest.json` persists `pixal3d-generation-smoke/1` JSON (validated with `scripts/validate_generation_manifest.py`); see `docs/generation-manifests/README.md`.
- **Adapter policy (stub):** `python3 scripts/check_adapter_policy.py` validates `docs/adapters/policy.example.json` (or `--policy PATH`) and prints `pixal3d-adapter-policy-check/1` JSON on stdout; see `docs/adapters/README.md`.
- **Adapter policy (runtime):** `/health` reports `adapter_policy_*` fields after `adapter_policy_runtime.py` runs at boot. Set `PIXAL3D_ADAPTER_POLICY_ENFORCE=1` only when the policy file lists enabled adapters that must match configured rembg Hub repos.
- **Generation run manifest (API):** successful `/generate_3d` responses include `generation_run` (`pixal3d-generation-run/1`). See `docs/generation-runs/README.md`.
- Before closing a task that changed runtime-facing files, run `python scripts/check_repo_parity.py` to confirm `github/main` matches HF `origin/main`; if drift is reported, sync with `git push origin main` or enable `HF_TOKEN` auto-sync.
- If deployment is unsafe because the working tree contains unrelated runtime changes, say so explicitly instead of silently publishing them.
